Laagan Adventure

Tour guide & travel company website for Zamboanga City, Philippines. Built on Next.js 15 + Sanity CMS.

🔗 Live: laagan-adventure.vercel.app 📦 Repo: JMS255/laagan-adventure 🛠 Framework: Next.js 15 App Router 📅 Last updated: May 2026 · Sprint 5 complete

Introduction

Project Overview

Laagan Adventure is a local tour operator website for Zamboanga City, Philippines. It lets travelers browse tour packages, view detailed itineraries, and inquire/book tours via a form or Messenger.

The site was designed to feel like a premium travel brand — think AutoCamp meets Trafalgar, adapted for a small local Filipino tour business. No online payment processing; booking is confirmed via Messenger.

Business

What it does

Showcases 5 tour packages, accepts booking inquiries, and drives users to Messenger for confirmation.

Audience

Who it's for

Local and visiting travelers in Zamboanga City looking to book guided tours to Santa Cruz Island and other destinations.

CMS

Content management

All tours, testimonials, gallery photos, and blog posts are managed through Sanity Studio at /studio.

Forms

Inquiry system

Contact form submits to Formspree. No backend required. Booking is confirmed via Messenger after inquiry.

Feature

Booking System

The booking system has three parts that share state via React Context (lib/booking-context.tsx):

Widget

BookingWidget.tsx

White card floating at the bottom of the hero. Fields: tour selector, date picker, guest counter. "Check Availability" opens the drawer.

Drawer

BookingDrawer.tsx

Slide-in panel from the right. Shows all 5 tour cards. Clicking a tour takes user to the contact page with pre-filled params.

Dropdown

TourDropdown (in Widget)

Custom-styled dropdown using createPortal to escape overflow:hidden on the hero section. Light theme, pink selection.

Calendar

DatePicker.tsx

Custom calendar component. No native browser UI. Smart above/below positioning. Pink selected date. Also uses createPortal.

Why createPortal? The hero section has overflow: hidden. Without portals, the dropdown and calendar get clipped. Both are rendered at document.body level with position: fixed coordinates from getBoundingClientRect().

Shared state flow

BookingProvider (app/layout.tsx)
  ├── Nav → "Book Now" button → openDrawer()
  ├── BookingWidget → sets tour/date/guests → openDrawer()
  └── BookingDrawer → reads state → links to /contact?tour=...&date=...&guests=...

Feature

Tour Detail Page

Inspired by Trafalgar's tour pages. Located at /tours/[slug]. Fully server-rendered with client islands for interactive parts.

Page sections (top to bottom)

Hero
Full-width 480px image with dark overlay. Shows destination badge, tour title, duration, price.
Sticky Tabs (TourTabs.tsx)
Sticks below the nav. "Your Tour" / "What's Included" / "Reviews". Active tab updates as you scroll via IntersectionObserver scrollspy.
Itinerary (2-column grid)
Pink location pin icons + time badge + activity text. Matches Trafalgar's sightseeing highlights layout.
Travel Highlights checklist
2-column grid of inclusions with green ✓ checkmarks. Pulls from Sanity inclusions field.
Google Maps embed
Renders if mapQuery is filled in Sanity. Uses Google Maps iframe (no API key needed). Shows the tour area.
FAQ Accordion (FAQAccordion.tsx)
6 default Zamboanga-specific FAQs built in. Override per tour by filling the faq field in Sanity.
Sticky Booking Card (sidebar)
Price, duration, "Book This Tour" → /contact, "Ask a Question" → Messenger. Trust badges. Sticks as you scroll.
What's Included section
Full-width pink-tinted section. 2 columns: inclusions (green ✓) and exclusions (red ✕).
Reviews
Up to 3 testimonials from Sanity. Shows empty state if none exist yet.
Similar Tours
3 other tour cards auto-pulled from Sanity, excluding the current tour. Uses SIMILAR_TOURS_QUERY.

Feature

Contact Page

Split-screen layout at /contact. Left panel: dark navy with tour highlights and contact info. Right panel: white inquiry form.

The form submits to Formspree (https://formspree.io/f/xpwzgwnn). Fields auto-populate from URL params: ?tour=...&date=...&guests=... — set when coming from the booking widget or drawer.

Booking flow: Widget → "Check Availability" → Drawer → "Book this tour" → Contact page with pre-filled tour/date/guests → Formspree submission → Owner replies via Messenger.

Sprint 5 — Emotional UX & Conversion ✅ Complete

Based on a deep live analysis of Trafalgar.com vs laagan-adventure.vercel.app, this sprint closes the emotional engagement gap. The goal: visitors should feel curious, excited, and safe within 10 seconds of landing. Right now we nail "safe" but miss "curious" and "excited." Every item below is targeted at that specific gap.

The core insight: Trafalgar triggers 5 psychological states — aspiration, social validation, trust, discovery, and FOMO. Laagan Adventure currently triggers 2 (trust, discovery). Sprint 5 adds the missing 3.

Traveler identity filter — 4 cards above the tour grid

Directly copied from Trafalgar's "Ways to Go" section — the highest-leverage UX change on the homepage. Implementation: client component with useState filter + tours tagged by audience in Sanity.

Promo Codes New

A simple client-side promo code engine. Codes live in lib/promoCodes.ts — no database or backend required. Supports both fixed (₱ off) and percent (% off) discounts.

Current active codes

CodeDiscountType
LAAGAN500₱500 offFixed

Adding a new code

Open lib/promoCodes.ts and add a line to the CODES object:

// lib/promoCodes.ts
const CODES = {
  'LAAGAN500':  { discount: 500, type: 'fixed',   label: '₱500 off your booking' },
  'SUMMER10':   { discount: 10,  type: 'percent',  label: '10% summer discount' },
  'MYCODE':     { discount: 300, type: 'fixed',   label: '₱300 off — new code' },  // ← add here
}

Commit and push. The new code is live instantly — no Sanity update needed.

Marketing uses: Post LAAGAN500 on Facebook during slow seasons. Give travel influencers a unique code to track referrals. Partner with hotels (e.g. GRANDASTORIA for guests at the Grand Astoria hotel).
